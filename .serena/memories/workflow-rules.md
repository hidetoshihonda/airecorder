# AIrecorder プロジェクト ワークフロールール

## 🚨 最重要ルール: ドキュメントファースト

**Issue の実装に着手する前に、必ず `web/docs/` 配下に分析レポートを作成すること。**

### 手順（厳守）

1. **Issue の内容を理解する**
2. **関連コードを全て読み込む**
3. **`web/docs/` に分析レポート（.md）を作成する** ← ここを絶対に飛ばさない
4. ユーザーにレビュー・承認を依頼する
5. 承認後、ブランチを作成して実装に着手する
6. PR を作成し、CI 通過後にマージする

### レポートの命名規則

- `Issue{番号}_{日本語の短い説明}_分析.md`
- 例: `Issue26_録音コントロールコンパクト化_分析.md`

### レポートの必須セクション

ReviewAAgent モードの出力フォーマットに従う:
1. エグゼクティブサマリー
2. アーキテクチャ概観
3. 重大バグ分析 / 設計上の問題
4. 依存関係マトリクス
5. 修正提案（Phase分け）
6. テスト戦略
7. 実装ロードマップ
8. リスクアセスメント
9. 結論（GO / NO-GO 判定）

### 過去の失敗

- Issue #26 で分析レポート作成を忘れ、ユーザーに2回指摘された
- 原因: コード読み込みに集中し、レポート出力ステップを忘却した
- 対策: **Issue 番号を受け取ったら、最初に「docs/ にレポートを作成します」と宣言する**
